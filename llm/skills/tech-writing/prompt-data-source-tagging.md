# Prompt：数据来源分类标注

## 用途
为技术文章中的性能数据（跑分、延迟、吞吐量等）标注来源性质，防止实测数据与估算数据混淆。

## 标注体系

### 数据来源类型
| 标注 | 含义 | 示例 |
|---|---|---|
| ✅ 硅后实测 | 已量产芯片的第三方或官方实测数据 | SpacemiT V100 SPEC2006 实测 |
| ⚠️ 厂商发布指标 | 芯片厂商自行发布，未经第三方验证 | Akeana 5300 SPEC2006/GHz |
| 〜 推算 | 从公开基准数据换算推导 | SPEC2017 × 9 → SPEC2006 |
| 〜 估算 | 从间接数据推算，有较大不确定性 | Graviton5 按 V3/V2 IPC 提升推算 |

### 芯片状态类型
| 标注 | 含义 |
|---|---|
| 量产 | 消费者/企业可购买 |
| 测试芯片 | Tape out 完成，开发板阶段 |
| IP | 仅授权 IP，需客户自行流片 |
| 流片验证中 | 已提交流片，结果未公开 |

## 表格标准格式

```markdown
| 芯片/IP | 指令集 | SPEC2006/GHz | 推算依据 | 来源 |
|---|---|---|---|---|
| 芯片名 | ISA | **数值** ✅硅后实测 | 原始数据说明 | [来源链接](url)（芯片状态）|
| 芯片名 | ISA | **数值** ⚠️厂商发布指标 | 换算说明 | [来源链接](url)（IP；状态）|
```

## Prompt

请检查以下跑分表格，对每一行输出：
1. 数据来源类型是否已标注（实测/厂商指标/推算/估算）
2. 芯片状态是否已注明（量产/测试芯片/IP）
3. 推算依据列是否完整（换算公式、测试条件、编译器参数）
4. 来源 URL 是否可访问

对于 SPEC2017 → SPEC2006 换算：
- 换算系数固定为 **×9**（非 ×10）
- 必须在推算依据列备注「SPEC2017 → SPEC2006，×9 换算」
- 单核 rate=1 数据需注明频率（GHz），方可计算 per-GHz 分值
