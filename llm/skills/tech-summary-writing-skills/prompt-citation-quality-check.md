# Prompt：引用质量检查

## 用途
发布前检查文章所有引用的完整性、一致性和可访问性。

## Prompt

请对以下文章执行引用质量检查，输出问题列表：

### 检查项 1：URL 可访问性
- 找出所有脚注中的 URL
- 标记明显错误的 URL（如截断、占位符、404 已知域名）
- 标记「链接待核实」的条目

### 检查项 2：正文与脚注一致性
- 找出正文中所有数字（秒、GHz、%、亿等）
- 对照脚注中对应的数字，检查是否一致
- 特别注意：同一数据在不同位置是否表述不同（如「15-40 s」vs「10-30 s」）

### 检查项 3：孤儿检测
- 找出有脚注定义但正文中没有引用的编号（孤儿定义）
- 找出正文中引用了但没有脚注定义的编号（缺失定义）

### 检查项 4：来源性质标注
- 检查每条数据是否注明来源性质：实测 / 厂商指标 / 推算 / 估算
- 检查硅验证状态是否注明：量产 / 测试芯片 / IP / 流片验证中

### 检查项 5：英文引文翻译
- 找出所有英文引文（blockquote 或 *斜体* 格式）
- 检查是否每条都附有中文译文

## 输出格式
```
[Must Fix] [^N] URL 失效：...
[Must Fix] 正文「X s」vs 脚注「Y s」不一致
[Should Fix] [^N] 缺少来源性质标注
[Info] 共 N 个脚注，N 个孤儿，N 个缺失定义
```
