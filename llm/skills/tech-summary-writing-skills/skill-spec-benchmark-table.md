# Skill：SPEC 跑分表格

## 适用场景
构建 RISC-V / ARM / x86 芯片 SPEC2006/GHz 横向对比表格。

---

## 标准表格格式

```markdown
| 芯片/IP | 指令集 | SPEC2006/GHz | 推算依据 | 来源 |
|---|---|---|---|---|
| 芯片名（型号） | RISC-V / ARM / x86 | **数值** ✅硅后实测 | 推算说明 | [来源](url)（芯片状态）|
```

### 排序规则
- 按 SPEC2006/GHz **从高到低**排列
- 估算区间（如 ~20~21）按下界排序

---

## 来源性质标注

| 标注符号 | 含义 | 使用条件 |
|---|---|---|
| ✅ 硅后实测 | 已量产芯片第三方实测 | 有可访问的原始跑分数据 |
| ⚠️ 厂商发布指标 | 厂商自行发布，未第三方验证 | 来源为厂商白皮书/官网 |
| （无标注）| 第三方推算/实测 | 独立机构测试，非厂商数据 |

### 芯片状态（写在来源列括号内）
- `量产` — 消费者/企业可购买
- `测试芯片` — tape out 完成，开发板阶段（需注明预计发货时间）
- `IP` — 仅授权 IP，需客户自行流片
- `硅验证，量产服务器` — 服务器量产形态

---

## SPEC 换算规则

### SPEC2017 → SPEC2006
- 换算系数：**×9**（非 ×10）
- 必须备注：「SPEC2017 → SPEC2006，×9 换算」
- 数据来源：需注明原始 SPEC2017 分值、频率（GHz）、rate=1 或 rate=N

### 换算公式
```
SPEC2006/GHz = SPEC2017_rate1_base ÷ 频率(GHz) × 9
```

### 示例（Ventana Veyron V2）
```
官方 SPEC2017 rate=1 绝对分 7 @ 3.2 GHz
÷ 3.2 = 2.187/GHz
× 9 = ~20 SPEC2006/GHz
```

---

## 交叉验证要求

单一来源数据需第二路径验证：

**AMD Zen5 示例**：
- 路径 1：jia.je 第三方实测 EPYC 9R45，SPECrate2017_int_base=10.3 @ 4.49GHz → 20.6/GHz
- 路径 2：Chips&Cheese Zen5 vs Zen4 +13.3% IPC → ~21/GHz
- 取均值或区间：~20~21/GHz

**Graviton 估算示例**：
- 基础：Graviton4 Phoronix 实测反推 ~22.7/GHz
- 提升：Arm Neoverse V3 vs V2 +12% IPC × 频率提升 3.1/2.8
- 结果：Graviton5 ~25/GHz（估算，无官方 SPEC 提交）

> **背景说明**：SPEC2006 已于 2018 年被 SPEC2017 取代，但 RISC-V 社区沿用 SPEC2006/GHz

## 独立核实清单

在填写芯片状态前，需独立核实：

- [ ] 是否有公开的量产发货记录（不仅是发布公告）
- [ ] 跑分数据是否来自可访问的原始链接
- [ ] 厂商宣称的跑分频率是否注明（避免「25分@不知道什么频率」）
- [ ] RISC-V 芯片是否为实际硅（IP 授权 ≠ 量产芯片）
